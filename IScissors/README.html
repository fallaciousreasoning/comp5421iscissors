The project has all the basic requirements specified in the grading sheets. However, there are a couple of problems with it, that I feel worth mentioning.

Firstly, I wrote my own simple masking algorithm, as I didn't feel like using some third party library for what was a relatively small part of the project. The algorithm works as follows:

  • For each point on the contour, color the pixel in the result image
  • Find the minimum point on the contour
  • Go up from this point till you find a blank pixel. This pixel must be inside the contour
  • Find all pixels with a depth first search that aren't colored and set them to the color in the original image. This is similar to paints 'fill' tool.
This works perfectly well unless the user defines a complex polygon (that is, a polygon that intersects itself), in which case only one of the areas of the polygon will be filled. For the purposes of this assignment I consider it adequate, as the focus is on the intelligent scissors algorithm.

Basic Use

  • Select 'Action->Start' to begin tracing the contour
  • Select 'Action->Stop' to pause
  • Select 'Action->Close' to close a contour or simply click on the first point
  • If you make a mistake you can undo with 'Edit-Undo'
  • You can reset the view or manually set the zoom level from the view menu
  • You can set the image mode to Default, Cost, Pixel or Sobel from the view menu
  • You can open a new image, clear the contour, save the contour or save an image mask from the file menu
  • Use the '+' and '-' buttons to zoom in and out
  • Right click and drag anywhere on the screen to pan around

The project also implements some of the bells and whistles mentioned in the marking sheet. They are as follows:

  • Optional blurring of the image. The user can add a Gaussian blur to the image with a radius between 1 and 10 (setting the radius to 0 removes the blur). The blur uses a sigma value of 1.4 and this cannot be changed through the UI. The main impact of the blur is that it reduces the impact of noise on the intelligent scissors program. At low levels, this means the program is less likely to snap to some non-existent edges. However, as the zoom gets higher, some real edges may get ignored, causing the scissors to only snap to the most pronounced edges. It is for this reason that the program has had its max zoom set to 10.
